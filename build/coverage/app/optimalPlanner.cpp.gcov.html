      27             :  * @file optimalPlanner.cpp
      28             :  * @brief  Astar Algorithm
      29             : A* Search Algorithm: At each step it picks the node according to a
      30             : value-‘f’ which is a parameter equal to the sum of two other parameters – ‘g’
      31             : and ‘h’. At each step it picks the node/cell having the lowest ‘f’, and process
      32             : that node/cell.
      33             : g cost = the movement cost to move from the starting point to a given square on
      34             : the
      35             : grid, following the path generated to get there.
      36             : h cost = the estimated movement cost to move from that given square on the grid
      37             : to
      38             : the final destination
      39             :  *
      40             :  * Here we are using Manhattan Distance for calculating h cost for each nodes
      41             :  *
      42             : Working:
      43             : 1. Add the starting square (or node) to the open list.
      44             : 
      45             : 2. Repeat the following:
      46             : 
      47             : A) Look for the lowest F cost square on the open list. We refer to this as the
      48             : current square.
      49             : 
      50             : B). Switch it to the closed list.
      51             : 
      52             : C) For each of the 8 squares adjacent to this current square …
      53             : 
      54             : If it is not walkable or if it is on the closed list, ignore it. Otherwise do
      55             : the following.
      56             : If it isn’t on the open list, add it to the open list. Make the current square
      57             : the parent of this square. Record the F, G, and H costs of the square.
      58             : If it is on the open list already, check to see if this path to that square is
      59             : better, using G cost as the measure. A lower G cost means that this is a better
      60             : path. If so, change the parent of the square to the current square, and
      61             : recalculate the G and F scores of the square. If you are keeping your open list
      62             : sorted by F score, you may need to resort the list to account for the change.
      63             : D) Stop when you:
      64             : 
      65             : Add the target square to the closed list, in which case the path has been found,
      66             : or
      67             : Fail to find the target square, and the open list is empty. In this case, there
      68             : is no path.
      69             : 3. Save the path. Working backwards from the target square, go from each square
      70             : to its parent square until you reach the starting square. That is your path.
      71             :
